# OASIS CRM v9.8.4

## 수정 목적

- v9.8.3에서 국민연금 기본·상세조회에 없는 신규취득자수와
  상실가입자수를 각각 0명으로 처리하던 문제를 수정합니다.
- 고용 증가를 전년 동월 가입자수와 최근 월 순취득으로 구분해 사용자가
  판단 기준을 선택할 수 있도록 합니다.
- 사용자가 이미 조회한 국민연금 페이지 구간을 다시 검색하지 않도록
  사용자별 검색 이력을 자동 저장하고 화면에 표시합니다.
- 개인사업자 후보를 원하는 업종으로 좁혀 찾을 수 있도록 합니다.
- 영업 실무 화면의 메뉴명과 위치를 단순화하고 결과 활용성을 높입니다.
- 저장된 업체마다 상담·통화 메모를 기록할 수 있도록 합니다.

## 변경 내용

- `고용 증가 판단 기준` 추가
  - 전년 동월 대비 가입자수 증가
  - 최근 월 신규취득자수-상실가입자수 증가
  - 고용 증가 필터 사용 안 함
- 전년 동월 기준 선택 시 동일 사업장의 12개월 전 기본조회와 상세조회를
  연결해 현재 가입자수와 비교
- 최근 월 기준 선택 시 국민연금 기간별 조회 API를 별도로 호출
- 고용자료가 없거나 동일 사업장을 확인하지 못한 경우 `0명`이 아닌
  `확인 불가`로 표시
- 검색 결과에 자료생성년월, 전년 가입자수, 전년 대비 증가,
  최근 월 신규취득·상실·순취득, 고용 판정과 자료상태 표시
- 저장 영업후보의 `source_data`에 적용 기준과 고용 판정 근거 보존
- 전년 대비 증가를 선택한 경우 초회 전화 스크립트에 전년 대비 가입자
  증가 근거 반영
- 좌측 메뉴 `영업후보DB`를 `DB발굴`로 변경
- `DB발굴` 메뉴를 모든 사용자의 `홈` 바로 다음 위치로 이동
- 기존 `영업후보DB` 세션을 `DB발굴`로 자동 전환
- 검색 완료 시 사용자별 지역·사업자 유형·페이지 구간 자동 저장
- `내 검색 페이지 이력`에서 최근 조회 구간과 다음 권장 페이지 표시
- 최근 검색 다음 페이지를 새 검색의 시작·종료 입력값에 자동 반영
- 개인사업자 후보 업종 필터 추가
  - 병원·의원
  - 음식점
  - 서비스업
  - 도소매업
  - 제조업
  - 건설업
  - 기타
- 업종 필터를 선택하지 않으면 기존처럼 전체 업종 검색
- 이번 발굴결과 엑셀 다운로드 추가
- 저장된 영업후보 엑셀 다운로드 추가
- 저장된 업체별 공용 메모 입력·저장 기능 추가
- 검색 이력 테이블이 아직 없거나 SQL을 실행하지 않은 경우에도
  기존 DB발굴·후보 조회 기능은 계속 동작하도록 호환 처리

## Supabase 추가 사항

- `oasis_prospect_companies.memo` 컬럼 추가
- `oasis_prospect_search_history` 테이블 및 조회용 인덱스 추가
- 검색 이력에 `growth_basis` 컬럼 추가
- 기존 테이블·행·컬럼은 삭제하지 않습니다.
- `supabase_v984_db_discovery.sql`을 SQL Editor에서 한 번 실행합니다.

## 보존 사항

- 기존 CRM·고객관리·상담일지·정책자금·고용지원금 기능을
  삭제하지 않았습니다.
- 기존 영업후보와 연락처 데이터는 유지됩니다.
- 모든 사용자가 이미 저장한 영업후보를 신규 검색에서 제외하는
  기존 전역 중복 방지 기능을 유지합니다.
- 일반전화와 휴대전화 모두 유효 연락처로 유지합니다.
- 국민연금 상세조회, 연락처 검색, 영업후보 저장 등 기존 기능은
  그대로 유지합니다.
- 기존 `.git` 연결을 수정하지 않습니다.
