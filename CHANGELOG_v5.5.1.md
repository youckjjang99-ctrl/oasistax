# CHANGELOG v5.5.1

## 녹음 상담일지 진행상태 표시 및 중복클릭 방지

### 작업 시작 방식 개선
- 생성 버튼 클릭 시 작업을 세션 대기열에 먼저 등록
- 즉시 화면을 다시 그려 진행상태 창을 먼저 표시
- 상태창이 표시된 뒤 녹취와 상담일지 생성을 시작

### 진행단계 표시
1. 녹음파일 확인
2. 캐시·전처리 조건 확인
3. 음성 녹취
4. 상담주제·고객 니즈 분석
5. 상담일지 초안 준비

### 중복 실행 방지
- 작업 대기 또는 실행 중 생성 버튼 비활성화
- 버튼 문구를 `상담일지 생성 중...`으로 변경
- 한 녹음파일이 중복으로 API 호출되는 문제 방지

### 오류 안내 개선
- API 잔액 부족 오류를 한국어로 안내
- HTTP 429 요청한도 오류 별도 안내
- 실패 후 버튼을 다시 사용할 수 있도록 상태 자동 초기화

### 데이터 보호
- 기존 고객DB·CRM·상담일지 데이터 변경 없음
- 기존 캐시 및 API 비용절감 기능 유지
