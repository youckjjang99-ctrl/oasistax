# CHANGELOG v7.3.0

## 정책자금 매칭 안정화

- 정책자금 매칭 실행 중 기업마당 API 직접 호출 제거
- 매칭 실행 중 K-Startup API 직접 호출 제거
- 매칭 실행 중 중진공 OpenAPI 직접 호출 제거
- 기업마당 연결 지연이 매칭 전체를 중단시키던 문제 해결
- 매칭은 내부 통합 정책DB만 조회하도록 변경
- 기존 내부 상시정책자금·고용지원금 자료 유지
- 새벽 3시 수집된 기업마당 자료를 내부 정책DB에 자동 반영
- K-Startup·중진공도 새벽 동기화 캐시에 저장 가능
- 외부 API 장애 시 마지막 정상 캐시 유지
- 매칭 화면에 최근 새벽 동기화 시각 표시
- 새벽 수집 실패 시 오류 전체문구 대신 마지막 자료 사용 안내
- 사용자 매칭 속도 및 안정성 개선

## 자동 업데이트

- GitHub Actions 매일 한국시간 새벽 3시 실행
- 기업마당: 기존 `collector.py`로 100건씩 분할 수집
- K-Startup·중진공: 선택적으로 GitHub Actions Secrets를 사용해 수집
- 수집자료를 GitHub 내부 JSON DB로 커밋
- Streamlit 앱은 배포된 내부 JSON DB를 Supabase/로컬 캐시에 병합

## 데이터 보호

- 기존 정책DB 삭제 없음
- 새벽 수집 실패 시 기존 정상자료 보존
- 고객DB·CRM·상담일지·정관·기업히스토리 영향 없음
- Supabase SQL 추가 실행 불필요
