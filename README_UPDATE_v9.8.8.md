# OASIS CRM v9.8.8 적용 안내

## 적용 기준

- v9.8.7이 적용된 프로젝트용 PATCH입니다.
- ZIP 내부 파일을 기존 `정책자금자동화` 폴더에 바로 덧붙입니다.
- 별도 상위 폴더 없이 압축을 풀고 `RUN_UPDATE_v9.8.8.bat`를 실행합니다.

## 수정 목적 · 파일 · 영향 범위

- `prospect_db_repository.py`: 영업후보 목록을 `owner_user_id`로 필터링하고,
  메모 수정도 해당 사용자 소유 업체에만 적용합니다. 저장 직전 공통 중복을
  다시 검사해 다른 사용자의 저장자료를 덮어쓰지 않습니다.
- `prospect_db_center.py`: 저장된 영업후보·연락처·엑셀·메모 화면을
  로그인 사용자 본인 저장 업체로 제한합니다.
- 기존 DB발굴의 전사 공통 중복제외는 유지합니다. 따라서 A가 저장한
  업체는 B에게 보이지 않으며, B가 새로 검색해도 다시 후보로 나오지
  않습니다.
- 기존 영업후보 데이터, 고객DB, 연락처DB, 검색 이력, Supabase 구조와
  `.git`은 삭제하거나 변경하지 않습니다.

## 적용 순서

1. ZIP 내용을 기존 프로젝트 폴더에 그대로 풉니다.
2. `RUN_UPDATE_v9.8.8.bat`를 실행합니다.
3. `UPDATE_OK`, `PY_COMPILE=OK`, `RUNTIME_SMOKE_TEST=OK`를 확인합니다.
4. 이번 버전은 Supabase SQL 실행이나 Railway 환경변수 추가가 없습니다.
5. 동봉된 Git 명령어로 GitHub main에 반영하고 Railway 배포가 완료되면
   서로 다른 사용자 계정으로 목록과 검색 결과를 확인합니다.

## 백업 · 롤백

- 적용 전 파일은 `_update_backups/before_v9.8.8`에 자동 백업됩니다.
- 컴파일 또는 스모크 테스트 실패 시 업데이트 파일을 자동 복원합니다.
- 수동 롤백이 필요하면 해당 백업 폴더의 파일을 프로젝트 루트에 덮어씁니다.
