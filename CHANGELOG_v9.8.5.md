# OASIS CRM v9.8.5

## 수정 목적

- 전년 동월 비교 API 경로가 900건을 조회해도 0건으로 필터링하던
  DB발굴 고용 증가 판정 문제를 수정합니다.

## 변경 내용

- 전년 동월 증가와 최근 월 순취득을 `통합 고용 증가 신호`로 변경
- 전년 자료 부재가 후보 제외 조건이 되지 않도록 수정
- 기간별 API의 최근 월 신규취득·상실가입자 수를 우선 조회
- 현재 가입자수를 Supabase 월별 스냅샷으로 자동 저장
- 전년 동월 값은 CRM 스냅샷이 축적된 경우에 함께 반영하도록 변경
- 고용 증가 신호는 전년 동월 증가 또는 최근 월 순취득 증가 중 하나만
  충족해도 후보로 인정
- `고용 증가 판단 기준` 선택 UI 삭제
- `시·군·구 법정동 코드`, `읍·면·동 법정동 코드` 입력 UI 삭제
- 검색 속도를 늦추던 후보별 과거 기본·상세 재조회를 제거

## 보존 사항

- 기존 DB발굴, 연락처 검색, 휴대전화 허용, 검색 이력, 업종 필터,
  엑셀 다운로드, 저장 영업후보 메모 기능을 유지합니다.
- 기존 고객과 모든 사용자가 저장한 영업후보 제외 기능을 유지합니다.
- 기존 Git 연결과 Supabase 데이터는 변경하지 않습니다.
