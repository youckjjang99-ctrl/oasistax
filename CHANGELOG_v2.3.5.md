# 정책자금자동화 v2.3.5 Change Log

## 변경 목적
- 크레탑 자동등록 화면 진입 시 지연되는 문제를 줄이기 위한 화면 렌더링 최적화

## 수정 파일
- app.py
- VERSION.txt
- CHANGELOG_v2.3.5.md

## 변경 내용
1. 기존 `st.tabs` 방식은 모든 탭 내용을 매번 실행하므로, 선택한 메뉴만 렌더링되는 메뉴 선택 방식으로 변경했습니다.
2. 크레탑 PDF는 같은 파일을 올린 상태에서 화면이 다시 실행되어도 매번 재분석하지 않도록 세션 캐시를 적용했습니다.
3. 크레탑 자동등록 탭 진입 시 매칭 실행, 실행이력, 담당자 통계, 회원 승인 관리 로직이 불필요하게 실행되지 않도록 분리했습니다.

## 기존 기능 영향 여부
- 기존 기능 삭제 없음
- 고객DB 업로드/매칭 실행 유지
- 크레탑 PDF 자동등록 유지
- 누적 고객DB 다운로드 유지
- 관리자 승인 기능 유지
