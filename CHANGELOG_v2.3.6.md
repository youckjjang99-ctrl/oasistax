# 정책자금자동화 v2.3.6 Change Log

## 변경 목적
- 크레탑 자동등록 메뉴 클릭 시 화면 전체가 반투명하게 보이는 UX 문제를 완화
- 탭/메뉴 전환만으로 무거운 파일 처리 작업이 실행되지 않도록 개선

## 수정 파일
- app.py
- VERSION.txt
- CHANGELOG_v2.3.6.md

## 변경 내용
1. Streamlit 재실행 중 기본 페이드 효과로 인해 화면이 반투명하게 보이는 현상을 CSS로 완화했습니다.
2. 크레탑 PDF는 업로드 즉시 분석하지 않고, 사용자가 `크레탑 PDF 분석하기` 버튼을 누를 때 분석되도록 변경했습니다.
3. 같은 PDF는 기존처럼 세션 캐시를 사용해 반복 분석하지 않습니다.
4. 최신 누적 고객DB 다운로드 파일은 화면 진입 시 바로 읽지 않고, 사용자가 다운로드 준비 버튼을 누를 때만 읽도록 변경했습니다.

## 기존 기능 영향 여부
- 기존 기능 삭제 없음
- 크레탑 PDF 자동등록 유지
- 누적 고객DB 저장 유지
- 누적 고객DB 다운로드 유지
- 매칭 실행/실행이력/관리자 기능 영향 없음
