# CHANGELOG v3.5.2

## 사업자등록번호 기준 중복 고객 자동 차단

### 수정 파일
- `app.py`
- `utils.py`
- `VERSION.txt`

### 주요 변경
1. 크레탑 자동등록 시 동일한 사업자등록번호가 누적 고객DB에 있으면 새 행을 추가하지 않습니다.
2. 중복 업체가 확인되면 저장 버튼을 비활성화하고 `이미 등록된 고객입니다`라고 표시합니다.
3. 기존의 `중복이어도 새 행으로 추가` 선택지를 제거했습니다.
4. 일반 고객DB 엑셀 업로드 시에도 기존 누적DB에 동일 사업자번호가 있는 행은 자동 제외합니다.
5. 하나의 업로드 파일 안에서 같은 사업자번호가 반복된 경우 첫 번째 행만 저장합니다.
6. 사업자등록번호가 없는 고객은 자동 판별이 불가능하므로 기존처럼 저장합니다.

### 기존 기능 영향
- 기존 누적 고객DB의 과거 중복 데이터는 자동 삭제하지 않습니다.
- 고객 삭제/중복관리 화면에서 기존 중복 데이터는 별도로 정리할 수 있습니다.
- CRM, 크레탑 추출, 매칭, 회원관리 기능은 유지됩니다.
