# OASIS CRM v9.4.3 업데이트 안내

## 수정 목적

고용종료된 직원이 현재 재직자로 집계되는 문제를 수정하고, 수도권 여부를
기업 주소에서 자동 판정하며, 추가정보 입력 중 화면 전체가 반복 갱신되는
현상을 제거합니다.

## 수정 파일

- `employee_status.py`
- `employment_support_2026.py`
- `enterprise_center.py`
- `system_precheck.py`
- `VERSION.txt`

## 영향 범위

- 직원현황의 검수표·재직자 집계
- 고용지원금의 수도권 자동판정
- 기업·채용계획 추가정보 입력 및 통합진단 실행 시점
- 다른 CRM 기능과 Supabase 구조에는 영향 없음

## 적용 방법

1. 압축 내용을 기존 `정책자금자동화` 폴더에 그대로 덮어 풉니다.
2. `RUN_UPDATE_v9.4.3.bat`를 실행합니다.
3. `UPDATE_OK`, `PY_COMPILE=OK`, `RUNTIME_SMOKE_TEST=OK`를 확인합니다.
4. 직원현황에서 기존 근로자 고용정보 Excel을 다시 업로드합니다.
5. 검수표에서 재직 2명·상실 2명을 확인하고 저장합니다.
6. 추가정보를 모두 입력한 후 `추가정보 확인 후 통합진단`을 누릅니다.

## 기존 저장자료 안내

v9.4.2에서 이미 저장된 직원자료에는 고용종료일이 유실되어 있어 자동
복원할 수 없습니다. 같은 Excel을 다시 업로드하면 종료정보가 포함된 새
버전이 저장되고 현재 재직자는 2명으로 집계됩니다. 이전 업로드 이력은
삭제하지 않습니다.

## 자동 백업과 롤백

- 적용 전 기존 파일은
  `_update_backups/v9.4.2_before_v9.4.3_날짜시간`에 백업됩니다.
- 적용 또는 실행 점검 실패 시 기존 파일로 자동 복구됩니다.
