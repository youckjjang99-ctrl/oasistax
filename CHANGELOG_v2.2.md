# CHANGELOG v2.2

## 변경 개요
- v2.1 회원가입 기능에 승인제 프로세스를 추가했습니다.
- 회원가입 직후 신규 계정은 `pending` 상태로 저장됩니다.
- 관리자 승인 전까지 신규 계정은 로그인할 수 없습니다.
- 관리자 계정은 승인 대기 회원을 승인 또는 거절할 수 있습니다.

## 수정 파일
- `auth.py`
- `app.py`
- `VERSION.txt`
- `CHANGELOG_v2.2.md`

## 상세 변경
### auth.py
- 신규 회원 기본 상태를 `pending`으로 변경했습니다.
- 관리자 계정은 `approved` 상태로 자동 생성/보정되도록 했습니다.
- v2.1에서 이미 생성된 기존 계정은 사용 중단을 막기 위해 `approved` 상태로 자동 보정되도록 했습니다.
- 승인 대기, 거절 계정은 로그인할 수 없도록 했습니다.
- 회원 승인/거절 및 전체 회원 조회 유틸 함수를 추가했습니다.

### app.py
- 관리자 로그인 시 `회원 승인 관리` 탭이 표시되도록 했습니다.
- 승인 대기 회원 목록, 승인 버튼, 거절 버튼을 추가했습니다.
- 전체 회원 현황 표를 추가했습니다.
- 상단 사용자 표시 영역에 관리자/회원 구분을 추가했습니다.

## 기존 기능 영향 여부
- 기존 매칭 실행, 실행이력, 담당자 통계, 누적 고객DB 다운로드 기능은 유지했습니다.
- 기존 v2.1 계정은 자동 승인 처리되므로 기존 사용자는 계속 로그인할 수 있습니다.
- 신규 회원은 관리자 승인 후에만 사용할 수 있습니다.

## 관리자 계정
- 기본 관리자 아이디는 `oasistax`입니다.
- 비밀번호는 기존 `APP_LOGIN_PW` 설정값을 따릅니다.
- 설정값이 없으면 기존 기본값 `1234`가 적용됩니다.
