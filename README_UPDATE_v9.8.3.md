# OASIS CRM v9.8.3 적용 안내

## 적용 기준

- v9.8.0, v9.8.1 또는 v9.8.2에서 바로 적용할 수 있는 누적 패치입니다.
- Railway 환경변수와 Supabase SQL을 추가할 필요가 없습니다.

## 적용 방법

1. 압축파일을 기존 `정책자금자동화` 폴더에 그대로 풉니다.
2. `RUN_UPDATE_v9.8.3.bat`를 실행합니다.
3. `UPDATE_OK`, `PY_COMPILE=OK`, `RUNTIME_SMOKE_TEST=OK`를 확인합니다.
4. 동봉된 Git 명령어로 GitHub main에 반영합니다.

## 검색 사용법

- 1~10페이지 검색 후 다음 검색은 시작 `11`, 종료 `20`을 입력합니다.
- `주식회사`, `개인사업자 후보`, `전체` 중 원하는 유형을 선택합니다.
- `순고용 증가 사업장만 표시`가 기본으로 체크되어 있습니다.
- 목표 업체 수를 채우거나 지정한 종료 페이지에 도달할 때까지 검색합니다.
- 정밀 검색을 복원했기 때문에 공개 API와 홈페이지 응답에 따라 시간이 오래 걸릴 수 있습니다.
- 화면에 표시되는 다음 검색 권장 페이지를 다음 시작 페이지로 사용합니다.

## 사용자 간 중복 제외

- 저장 영업DB는 사용자 공동 DB입니다.
- A 사용자가 저장한 업체는 이후 B 사용자의 신규 검색결과에서 자동 제외됩니다.
- 기존 저장자료는 삭제하지 않습니다.

## 개인사업자 분류

- 국민연금 API에는 법적 사업자 유형이 별도 필드로 제공되지 않습니다.
- 주식회사·유한회사·농업회사법인·각종 법인·협동조합 표기가 없는 사업장을 `개인사업자 후보`로 표시합니다.

## 백업과 롤백

- 적용 전 파일은 `_update_backups/before_v9.8.3`에 자동 백업됩니다.
- 적용 또는 테스트 실패 시 자동 롤백됩니다.
