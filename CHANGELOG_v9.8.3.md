# OASIS CRM v9.8.3

## 수정 목적

- 120초 제한 때문에 목표 업체 수를 채우지 못하는 검색 흐름을 이전의 정밀검색 방식으로 복원했습니다.
- 조회 페이지와 사업자 유형을 사용자가 직접 선택하도록 개선했습니다.
- 모든 사용자가 저장한 영업후보를 전역 중복 제외합니다.
- 순고용 증가 사업장을 기본 검색대상으로 적용했습니다.

## 변경 내용

- 검색 시작 페이지와 종료 페이지 직접 입력
- 한 번에 최대 100페이지 범위 조회
- 다음 검색 권장 시작 페이지 표시
- 사업자 유형: `주식회사`, `개인사업자 후보`, `전체`
- 개인사업자 후보는 법인표기가 없는 국민연금 사업장으로 분류
- `순고용 증가 사업장만 표시` 기본 활성화
- 순고용증가 계산: 신규취득자수 - 상실가입자수
- 순고용 증가가 없는 업체는 기본 검색결과에서 제외
- 120초 강제 종료 제거
- 국민연금 조회 30초 및 재시도 2회 복원
- 공식 홈페이지 후보 3개, 후보별 최대 4페이지 정밀검색 복원
- 일반전화와 휴대전화 모두 검색·표시·저장
- 저장 영업후보를 사용자 구분 없이 전체 테이블에서 중복 제외
- 저장목록을 순고용 증가, 가입자 순으로 정렬

## 보존 사항

- 기존 CRM·고객관리·상담일지·정책자금·고용지원금 기능을 삭제하지 않았습니다.
- 기존 고객DB·영업DB·Supabase 스키마를 변경하지 않았습니다.
- 기존 `.git` 연결을 수정하지 않습니다.
