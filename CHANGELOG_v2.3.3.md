# 정책자금자동화 v2.3.3 Change Log

## 수정 목적
크레탑 PDF 자동등록 후 누적 고객DB가 실제로 갱신되지 않은 것처럼 보이거나, 구버전 누적DB 형식이 그대로 다운로드되는 문제를 보정했습니다.

## 수정 파일
- app.py
- utils.py
- VERSION.txt
- CHANGELOG_v2.3.3.md

## 변경사항
1. 누적 고객DB 다운로드 직전에 기존 누적DB를 최신 고객DB 템플릿 형식으로 강제 변환합니다.
2. 크레탑 PDF 저장 후 즉시 누적 고객 수를 계산해 저장 성공 메시지를 표시합니다.
3. 기존 v2.3~v2.3.1에서 생성된 `고객DB누적` 단일 시트 파일도 `고객DB` 시트 및 전체 템플릿 시트 구조로 변환합니다.
4. `상시정책자금DB`, `고용지원금DB`, `코드표`, `사용가이드` 시트를 유지합니다.
5. `누적저장일시`, `회원ID`, `담당자명` 컬럼은 고객DB 다운로드 파일에서 제거합니다.

## 기존 기능 영향
- 기존 매칭 기능 유지
- 회원별 누적DB 기능 유지
- 크레탑 PDF 자동등록 기능 유지
- 기존 구버전 누적DB는 삭제하지 않고 최신 형식으로 변환
