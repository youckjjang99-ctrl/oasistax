# 정책자금자동화 v2.3.4 Change Log

## 수정 목적
- 크레탑 PDF 자동등록 후 저장은 되었지만, 같은 실행 흐름에서 기존 다운로드 버튼이 과거 누적 고객DB를 읽어 사용자가 추가되지 않은 것처럼 보이는 문제를 수정했습니다.

## 수정 파일
- app.py
- VERSION.txt
- CHANGELOG_v2.3.4.md

## 변경 사항
1. 크레탑 PDF 저장 완료 후 `st.rerun()`을 실행하여 앱 전체가 최신 누적DB 파일을 다시 읽도록 수정했습니다.
2. 재실행 후에도 저장 성공/미저장 메시지가 유지되도록 `st.session_state` 기반 알림 영역을 추가했습니다.
3. 크레탑 자동등록 탭에 최신 누적 고객DB 다운로드 버튼을 추가해 저장 직후 바로 확인할 수 있게 했습니다.

## 기존 기능 영향
- 기존 고객DB 업로드/매칭 기능에는 영향이 없습니다.
- 누적 고객DB의 기존 서식/시트 유지 로직은 그대로 유지됩니다.

## Streamlit 재배포
- GitHub에 수정 파일 업로드 후 Streamlit 재배포가 필요합니다.
