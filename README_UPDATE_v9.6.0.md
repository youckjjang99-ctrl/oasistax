# OASIS CRM v9.6.0 업데이트 안내

## 수정 목적

국민연금 사업장 API에서 서울·경기 사업장을 최대 100건씩 안전하게
조회하고, 기존 고객을 제외한 뒤 선택한 업체만 별도의 영업후보DB에 저장합니다.

## 수정 파일

- `app.py`
- `public_data_api.py`
- `prospect_db_center.py`
- `prospect_db_repository.py` 신규
- `supabase_v960_prospect_db.sql` 신규
- `VERSION.txt`

## 적용 방법

1. ZIP 내용을 기존 `정책자금자동화` 폴더에 바로 덮어 풉니다.
2. `RUN_UPDATE_v9.6.0.bat`를 실행합니다.
3. `UPDATE_OK`, `PY_COMPILE=OK`, `RUNTIME_SMOKE_TEST=OK`를 확인합니다.
4. Supabase → SQL Editor → New query로 이동합니다.
5. 프로젝트 폴더의 `supabase_v960_prospect_db.sql` 내용을 붙여 넣고
   `Run`을 한 번 실행합니다.
6. 동봉된 Git 명령어로 GitHub `main`에 업로드합니다.
7. Railway 배포 완료 후 관리자 계정으로 로그인합니다.
8. `영업후보DB` → `테이블 연결 확인`을 누릅니다.
9. 지역·페이지·조회건수를 정하고 `사업장 미리보기 수집`을 누릅니다.
10. 저장할 업체를 선택한 뒤 `영업후보DB에 저장`을 누릅니다.

## 데이터 보호

- 기존 고객DB와 CRM 데이터는 수정하지 않습니다.
- 기존 고객과 사업자등록번호가 같은 업체는 후보에서 제외합니다.
- 한 번의 API 조회는 최대 100건입니다.
- 서울·경기 외 사업장은 저장 대상에서 제외합니다.
- 동일한 공공데이터 사업장은 다시 저장해도 중복 생성되지 않습니다.

## 자동 백업과 롤백

- 적용 전 기존 파일은
  `_update_backups/v9.5.0_before_v9.6.0_날짜시간`에 백업됩니다.
- 적용 또는 검사 실패 시 기존 파일을 복원하고 신규 파일을 제거합니다.
- `.git`과 기존 Git 연결은 변경하지 않습니다.
