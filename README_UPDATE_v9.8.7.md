# OASIS CRM v9.8.7 적용 안내

## 적용 기준

- v9.8.6이 적용된 프로젝트용 PATCH입니다.
- ZIP 내부 파일을 기존 `정책자금자동화` 폴더에 바로 덧붙입니다.
- 별도 상위 폴더 없이 압축을 풀고 `RUN_UPDATE_v9.8.7.bat`를 실행합니다.

## 수정 목적 · 파일 · 영향 범위

- `app.py`: 고객관리의 기업 기본정보와 AI 기업분석 카드에 설립연차를
  표시합니다. 기존 설립일/설립년도 값만 읽어 계산하므로 DB 구조 변경은
  없습니다.
- `public_data_api.py`: 국민연금 사업장 조회 지역코드를 전국 17개 시·도로
  확장하고 주소 기반 지역 판별도 전국 기준으로 보완합니다.
- `prospect_db_center.py`: DB발굴 안내 문구를 전국 수집 기준으로 맞춥니다.
- 기존 고객DB, 설립일 입력 화면, 서울·경기 DB발굴, 연락처·고용 증가
  필터, 영업후보 저장과 `.git`은 삭제하거나 변경하지 않습니다.

## 설립연차 표시 기준

- 설립일 또는 설립년도에서 연도를 읽어 `현재연도 - 설립연도 + 1`로
  계산합니다.
- 3년차 이하: 신규기업 / 4~7년차: 성장초기 / 8년차 이상: 기존기업
- 설립일이 누락되면 `확인 필요`로 표시하며, 임의의 설립일을 생성하지
  않습니다.

## 적용 순서

1. ZIP 내용을 기존 프로젝트 폴더에 그대로 풉니다.
2. `RUN_UPDATE_v9.8.7.bat`를 실행합니다.
3. `UPDATE_OK`, `PY_COMPILE=OK`, `RUNTIME_SMOKE_TEST=OK`를 확인합니다.
4. 이번 버전은 Supabase SQL 실행이나 Railway 환경변수 추가가 없습니다.
5. 동봉된 Git 명령어로 GitHub main에 반영하고 Railway 배포가 완료되면
   고객관리와 DB발굴을 확인합니다.

## 백업 · 롤백

- 적용 전 파일은 `_update_backups/before_v9.8.7`에 자동 백업됩니다.
- 컴파일 또는 스모크 테스트 실패 시 업데이트 파일을 자동 복원합니다.
- 수동 롤백이 필요하면 해당 백업 폴더의 파일을 프로젝트 루트에 덮어씁니다.
