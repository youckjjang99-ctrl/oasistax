# OASIS CRM v9.7.4 패치 안내

## 적용 방법

1. 이 ZIP의 내용을 `정책자금자동화` 프로젝트 폴더에 그대로 압축 해제합니다. 최상위 폴더가 추가되지 않습니다.
2. 프로젝트 루트에서 `RUN_UPDATE_v9.7.4.bat`를 실행합니다.
3. `UPDATE_OK`, `PY_COMPILE=OK`, `RUNTIME_SMOKE_TEST=OK`가 나오면 완료입니다.
4. Railway는 GitHub `main` 푸시 후 자동 배포가 끝난 뒤 새로고침합니다.

## 동작 기준

- 영업후보 신규 수집은 명칭에 `주식회사`, `(주)`, `㈜` 표기가 있고 공개 대표전화가 확인된 업체만 대상으로 합니다.
- 농업회사법인과 유한회사는 물론 합자·합명·영농조합·사단·재단법인도 제외합니다.
- 기존 저장 행은 삭제하지 않습니다. 비대상 행은 영업후보 화면에서만 숨깁니다.
- 대표전화가 없으면 카카오 로컬(상호+주소 → 상호 단독) → 승인 인허가 API → 네이버 웹검색 → 공식 홈페이지 순서로 공개 사업용 연락처를 확인합니다.
- 공개 대표전화가 확인되지 않은 업체는 영업후보 표와 저장목록에 표시하지 않습니다. 원본 수집 데이터와 기존 저장 데이터는 삭제하지 않습니다.
- KIPRIS API는 호출하지 않으며 `KIPRIS_API_KEY`도 더 이상 필요하지 않습니다.
- 실행 BAT 파일은 Windows 인코딩 영향이 없도록 영문 명령만 사용합니다.

## DB / Railway

- 새 Supabase SQL 실행은 필요 없습니다.
- 기존 Railway 변수인 카카오·네이버·공공데이터포털 키를 그대로 사용합니다.
- KIPRIS 키가 Railway에 남아 있어도 앱은 사용하지 않습니다. 원하면 Railway Variables에서 수동 삭제할 수 있습니다.

## 롤백

업데이트 전 파일은 `_update_backups/before_v9.7.4/`에 자동 백업됩니다. 문제가 있으면 그 폴더의 파일 여섯 개를 프로젝트 루트에 덮어쓰면 이전 상태로 되돌릴 수 있습니다.
