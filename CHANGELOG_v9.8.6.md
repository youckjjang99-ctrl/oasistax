# OASIS CRM v9.8.6

## 수정 목적

- 개인사업자 재조회 시 저장된 영업후보가 다시 보일 수 있는 중복제외와
  월별 가입자 스냅샷 저장 오류를 수정합니다.

## 변경 내용

- 동일 요청 안에서 같은 월·같은 사업장 스냅샷 행을 먼저 통합하여
  Supabase `ON CONFLICT ... cannot affect row a second time` 오류 방지
- 국민연금에서 마스킹한 개인사업자 번호도 상호·주소를 함께 사용해
  사업장 단위로 구분
- 모든 사용자가 저장한 영업후보를 `source_key`, 완전한 사업자번호,
  상호·주소 중 하나가 일치하면 재조회 결과에서 제외
- 스냅샷 저장 오류와 기존 DB 중복확인 오류를 별도 안내로 분리

## 보존 사항

- 기존 DB발굴, 연락처 검색, 휴대전화 허용, 검색 이력, 업종 필터,
  엑셀 다운로드, 저장 영업후보 메모 및 통합 고용 증가 신호를 유지합니다.
- 기존 고객과 모든 사용자가 저장한 영업후보 제외 기능을 유지합니다.
- 기존 Git 연결과 Supabase 데이터는 변경하지 않습니다.
