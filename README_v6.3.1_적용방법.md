# 정책자금자동화 v6.3.1 적용방법

1. 이 ZIP을 기존 `정책자금자동화` 프로젝트 루트 폴더에 압축 해제합니다.
2. `RUN_V6.3.1_UPDATE.bat`를 실행합니다.
3. 업데이트 성공 메시지를 확인합니다.
4. Supabase SQL Editor에서 `supabase_v631_upgrade.sql`을 한 번 실행합니다.
5. 앱을 재시작한 뒤 기존 녹음파일을 다시 업로드해 `클라우드 캐시` 재사용 여부를 확인합니다.

## GitHub 반영
```bash
git status
git add .
git commit -m "v6.3.1 녹음파일 클라우드 캐시 복원 및 중복분석 방지"
git push origin main
```

## 참고
- SQL을 실행하지 않아도 기존 로컬 캐시는 정상 작동합니다.
- SQL 실행 후 새로 생성되는 녹취록과 상담일지는 Supabase 캐시에도 저장됩니다.
- 과거에 음성파일만 저장되고 녹취/상담일지 캐시가 저장되지 않은 건은 최초 1회 재분석이 필요할 수 있습니다.
