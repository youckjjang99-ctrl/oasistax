# OASIS CRM v9.8.0 변경 내역

## 잘못된 전화번호 차단

- `20260723`, `20162015`처럼 날짜·문서번호로 보이는 8자리 숫자를 전화번호로 인정하지 않습니다.
- 8자리 번호는 `15xx`, `16xx`, `18xx` 대표번호만 허용합니다.
- 국내 지역번호·인터넷전화·휴대전화 형식에 맞지 않는 숫자는 후보표와 저장목록에서 제외합니다.
- 기존 DB의 잘못된 숫자는 삭제하지 않고 사용자 목록에서 숨깁니다.

## 목표 수량 자동 수집

- 사용자는 지역과 목표 수량을 고른 뒤 `연락 가능한 성장기업 30개 찾기`만 누릅니다.
- 한 페이지에서 30개가 확보되지 않으면 설정한 최대 페이지까지 자동으로 이어서 조회합니다.
- 주식회사만 상세조회하며, 기존 고객DB와 기존 영업후보DB에 저장된 업체는 자동 제외합니다.
- 대표전화가 확인된 업체만 결과표와 저장대상에 표시합니다.

## 고용 증가 우선순위

- 국민연금 데이터의 `신규취득자수-상실가입자수`가 양수인 최근 순고용 증가 사업장을 먼저 찾습니다.
- 최근 순고용 증가 업체에서 목표 수량이 부족한 경우에만 나머지 업체를 확인합니다.
- 국민연금 공개데이터의 신규·상실 인원은 전달 고지대상자 대비 값이므로 전년 대비라고 표기하지 않습니다.

## 검색시간 단축

- 기본조회 단계에서 주식회사와 미저장 업체만 남긴 뒤 상세조회합니다.
- 카카오·승인 인허가 API를 빠르게 먼저 확인합니다.
- 빠른 조회로 부족한 수량만 네이버·공식 홈페이지 정밀검색을 진행합니다.
- 상세조회와 연락처 확인은 제한된 병렬 처리로 수행합니다.

## UI 재설계

- 일반 화면을 `조건 선택 → 성장기업 찾기 → 검토·저장` 흐름으로 단순화했습니다.
- API 키, 연결 테스트, Supabase SQL 다운로드는 일반 영업후보 화면에서 숨겼습니다.
- 기술 설정과 연결 테스트는 관리자 `시스템 관리` 페이지로 이동했습니다.

## 데이터 영향

- 기존 고객, 영업후보, 연락처 및 Supabase 테이블 구조를 변경하거나 삭제하지 않습니다.
- 신규 SQL 실행은 필요하지 않습니다.
