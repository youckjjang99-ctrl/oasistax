# OASIS CRM v9.8.0 패치 안내

## 적용 방법

1. ZIP의 파일과 `payload` 폴더를 기존 `정책자금자동화` 폴더에 그대로 압축 해제합니다.
2. 프로젝트 루트에서 `RUN_UPDATE_v9.8.0.bat`를 실행합니다.
3. `UPDATE_OK`, `PY_COMPILE=OK`, `RUNTIME_SMOKE_TEST=OK`를 확인합니다.
4. 동봉된 Git 명령어로 GitHub `main`에 올리고 Railway 배포 완료를 기다립니다.

## 사용자 화면

- 영업후보DB에서 지역, 목표 업체 수, 최소 가입자 수를 선택합니다.
- `연락 가능한 성장기업 30개 찾기`를 누르면 다음 순서로 자동 진행합니다.
  1. 국민연금 사업장 기본조회
  2. 주식회사·미저장 업체만 상세조회
  3. 최근 순고용 증가 업체 우선 정렬
  4. 카카오·인허가 대표전화 빠른 확인
  5. 부족한 수량만 네이버·공식 홈페이지 정밀 확인
  6. 유효한 대표전화가 있는 업체만 표시
- 다시 검색하면 이전 검색의 다음 페이지부터 이어서 찾고, 영업후보DB에 이미 저장된 업체는 제외합니다.

## 고용 증가 기준

현재 국민연금 공개 API가 제공하는 `신규취득자수`와 `상실가입자수`는 전달 고지대상자 대비 월간 변동입니다. 따라서 v9.8.0에서는 `신규취득자수-상실가입자수`가 양수인 업체를 `최근 순고용 증가`로 우선 처리합니다. 정확한 전년 대비는 12개월 전 스냅샷이 확보된 이후 별도 기능으로 판정해야 합니다.

## 관리자 화면

- 국민연금·카카오·네이버·인허가 API 연결 점검은 `시스템 관리` 하단의 `영업후보 데이터 연결 관리`에서 실행합니다.
- Supabase 영업후보·연락처 SQL 다운로드도 같은 관리자 화면에 있습니다.
- 신규 SQL 실행은 필요하지 않습니다.

## 롤백

업데이트 전 파일은 `_update_backups/before_v9.8.0/`에 자동 백업됩니다. 문제가 생기면 해당 폴더의 기존 파일을 프로젝트 루트에 덮어쓰면 이전 상태로 복원됩니다. 새로 추가된 `prospect_collection_service.py`는 롤백 시 자동 제거됩니다.
