# v5.5.1 적용방법

이번 패치는 녹음 상담일지 생성 버튼을 눌러도 진행상태가 보이지 않는 문제를 개선합니다.

## 적용

1. 압축 안의 파일을 기존 정책자금자동화 폴더에 풉니다.
2. `RUN_V551_FIX.bat`를 실행합니다.
3. GitHub에 반영합니다.

```powershell
git add .
git commit -m "v5.5.1 상담일지 진행상태 중복클릭 방지"
git push
```

4. Streamlit Cloud를 Reboot합니다.

## 정상 작동 화면

버튼을 누르면 즉시 다음 상태창이 표시됩니다.

```text
녹음파일 상담일지를 생성하고 있습니다.

1단계 · 녹음파일 확인
2단계 · 캐시·전처리 조건 확인
3단계 · 음성 녹취
4단계 · 상담주제와 고객 니즈 분석
5단계 · 상담일지 초안 준비
```

작업 중에는 버튼이 `상담일지 생성 중...`으로 바뀌고 비활성화되어 중복 클릭할 수 없습니다.

API 잔액 부족이나 요청한도 오류가 발생하면 원인을 한국어로 표시하고 버튼을 다시 사용할 수 있게 자동 초기화합니다.
